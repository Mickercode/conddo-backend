## 🐛 Bug 1 — `/brand-packages/usage` 500 for unsubscribed tenants

```
GET /api/v1/brand-packages/usage  →  500 INTERNAL_ERROR
```

The endpoint should respond cleanly for a tenant without a
subscription — either 200 with empty counts, or 404 with envelope.
Today it crashes with an unhandled exception.

**Why this matters**: the FE Brand Packages page calls
`/brand-packages/usage` to show the per-offering quota bars on the
current-subscription card. For unsubbed tenants, the FE soft-falls
to empty counts via QueryBoundary's 5xx → empty branch, so users
don't see a crash — but the BE log fills with INTERNAL_ERROR noise
every page load.

**Expected shape on unsubscribed**:
```json
{ "success": true, "data": null }
```
or
```json
{ "success": true, "data": { "counts": {}, "periodStart": null, "periodEnd": null } }
```

The FE handles both gracefully (uses `?? {}` on the counts).

## 🐛 Bug 2 — `/public/{slug}/auth/login` 500 on bad input

```
POST /api/v1/public/seb-bayorpharmaceutical/auth/login  →  500 INTERNAL_ERROR
(no body, valid Site-API-Key — should be 400 BAD_REQUEST)
```

Validation should reject a missing-fields login attempt with
`400 BAD_REQUEST` + a field-error envelope. Today it crashes.

**Why this matters**: this is the first endpoint Seb&Bayor's dev
will hit when wiring up the login form per
[STUDIO_DEV_ONBOARDING_SEB_BAYOR.md](./STUDIO_DEV_ONBOARDING_SEB_BAYOR.md)
§2 Test 6. A 500 looks like Conddo is broken; a 400 with
`{"email": "Email is required", "password": "Password is required"}`
tells the dev exactly what to fix.

Likely fix: add `@Valid @RequestBody` to the controller method and
ensure the `AuthLoginRequest` record has `@NotBlank @Email`
+ `@NotBlank` annotations on email + password.

The other public-auth endpoints (register, forgot-password,
reset-password) all behave correctly (401 with envelope on bad
Site-API-Key). Only `/auth/login` has the bug.
